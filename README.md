# کتابخانه شب — bookir.site

فروشگاه کتاب الکترونیک با تم تاریک سینمایی، ۴ دسته‌بندی کتاب، سبد خرید، ثبت‌نام/ورود دمو، و زیرساخت آماده برای پرداخت زرین‌پال (فعلاً خاموش).

## وضعیت فعلی سایت

- **۱۵ کتاب** در ۴ دسته: رمان، مهندسی، حقوقی، روان‌شناسی
- رمان **«شاید او من باشد»** با برچسب «به‌زودی» — بدون قیمت و دکمه‌ی خرید
- **پرداخت آنلاین غیرفعال است** (چون احراز هویت زرین‌پال هنوز تایید نشده). به‌جایش همه‌جا یک پیام «به‌زودی فعال می‌شود» نشان داده می‌شود و کاربر می‌تواند کتاب را به سبد خرید اضافه کند.
- صفحه‌ی **دسته‌بندی** (`/categories`) کتاب‌ها را بر اساس موضوع گروه‌بندی می‌کند.

---

## فعال کردن پرداخت (وقتی زرین‌پال تایید شد)

فقط یک خط را تغییر بدهید:

**`lib/config.js`**
```js
export const PAYMENTS_ENABLED = true; // به‌جای false
```

و در Environment Variables (چه لوکال چه Netlify) این‌ها را ست کنید:
- `ZARINPAL_MERCHANT_ID` = کد پذیرنده‌ی واقعی‌تان
- `ZARINPAL_SANDBOX` = `false`
- `JWT_SECRET` = یک رشته‌ی تصادفی طولانی
- `SITE_URL` = آدرس نهایی سایت (`https://bookir.site`)

همین. بقیه‌ی کد (صفحه‌ی پرداخت، بازگشت از درگاه، دانلود امن) از قبل آماده است و نیازی به تغییر ندارد.

## اجرا روی سیستم خودتان

```bash
npm install
cp .env.example .env
npm run dev
```

سایت روی `http://localhost:3000` بالا می‌آید.

---

## دیپلوی روی Netlify

۱. کد را در یک ریپازیتوری GitHub آپلود کنید (یا ریپوی قبلی‌تان را با این فایل‌ها آپدیت کنید).

۲. در [netlify.com](https://netlify.com) روی **Add new site → Import an existing project** بزنید و ریپو را انتخاب کنید.

۳. تنظیمات build از `netlify.toml` خودکار خوانده می‌شود.

۴. چون پرداخت فعلاً خاموش است، نیازی نیست همین الان متغیرهای زرین‌پال را ست کنید — می‌توانید مستقیم Deploy بزنید.

۵. وقتی دامنه (bookir.site) به Netlify وصل شد (بخش **Domain settings**)، DNS باید propagate بشود — همان چیزی که گفتید «در حال تابیدن» است؛ معمولاً چند ساعت تا یک روز طول می‌کشد.

## اضافه یا ویرایش کتاب

فایل `data/books.js` را باز کنید. هر کتاب یک آبجکت با این فیلدهاست:

```js
{
  id: "یک-اسلاگ-یکتا",
  title: "عنوان کتاب",
  author: "نام نویسنده",
  price: 50000,
  category: "novel" | "engineering" | "legal" | "psychology",
  description: "توضیح کوتاه",
  cover: "/covers/نام-فایل.svg",
  featured: true,      // در صفحه اصلی و بالای دسته نشان داده می‌شود
  published: true,     // false = «به‌زودی»، بدون قیمت/خرید
  fileContent: "..."   // محتوای فایل دمو - برای فایل واقعی، راهنمای زیر را ببینید
}
```

برای جلد کتاب، یک SVG در `public/covers/` بسازید یا عکس واقعی جلد را همان‌جا بگذارید (فرمت jpg/png هم قابل استفاده است، فقط مسیر `cover` را عوض کنید).

---

## اضافه کردن فایل واقعی کتاب (PDF/EPUB)

الان هر کتاب یک متن نمونه (`fileContent`) دارد. برای فایل واقعی:
1. فایل را در پوشه‌ی `private-files/` بگذارید (این پوشه در `public/` نیست، پس مستقیم در دسترس عموم نیست).
2. در `pages/api/download/[token].js`، به‌جای فرستادن متن، فایل واقعی را از دیسک بخوانید و به‌صورت باینری برگردانید — هروقت خواستید همین‌جا کمکتان می‌کنم.

## نکات مهم درباره‌ی بخش‌های دمو

- **ثبت‌نام/ورود**: فقط در مرورگر شما ذخیره می‌شود (`localStorage`)، رمز عبور رمزنگاری نمی‌شود. برای نسخه‌ی نهایی باید با بک‌اند امن جایگزین شود.
- **سبد خرید**: هم در مرورگر ذخیره می‌شود؛ هر کاربر روی مرورگر خودش سبد جدا دارد.
- **سفارش‌ها در دیتابیس ذخیره نمی‌شوند** — اطلاعات سفارش در یک توکن امضاشده (JWT) نگه داشته می‌شود، بدون نیاز به دیتابیس.

## قدم بعدی که پیشنهاد می‌کنم
- وقتی زرین‌پال تایید شد: خبرم کنید تا با هم فعالش کنیم و تست نهایی بزنیم
- وصل کردن یک دیتابیس واقعی (مثلاً Supabase رایگان) برای گزارش فروش و سفارش‌ها
- ثبت‌نام/ورود واقعی روی سرور
- آپلود فایل واقعی کتاب‌ها به‌جای متن نمونه

## پنل مدیریت (`/admin`)

با این پنل می‌توانید بدون دست زدن به کد، کتاب اضافه/ویرایش/حذف کنید.

### چطور کار می‌کند؟
- آدرس: `bookir.site/admin`
- رمز عبور از متغیر محیطی `ADMIN_PASSWORD` خوانده می‌شود — **حتماً در تنظیمات Netlify این را به یک رمز قوی تغییر دهید** (پیش‌فرض کد فقط برای تست محلی است و ناامن است).
- بعد از ورود، لیست کتاب‌ها را می‌بینید و می‌توانید افزودن/ویرایش/حذف کنید.

### داده‌ها کجا ذخیره می‌شوند؟
- **روی Netlify (سایت واقعی شما):** با Netlify Blobs، به‌صورت کاملاً خودکار و بدون نیاز به تنظیم اضافه. هر تغییری که در پنل بدهید، فوراً روی سایت اعمال می‌شود، بدون نیاز به دیپلوی دوباره یا commit در گیت‌هاب.
- **روی سیستم خودتان (npm run dev):** چون Netlify Blobs بیرون از محیط Netlify در دسترس نیست، خودکار در یک فایل محلی (`.data/books.json`) ذخیره می‌شود تا بتوانید پنل را تست کنید. این فایل در `.gitignore` است و آپلود نمی‌شود.

### نکته مهم
کاتالوگ اولیه (همان کتاب‌هایی که الان می‌بینید) در `data/books.js` تعریف شده و فقط **یک‌بار** به‌عنوان مقدار اولیه در Blobs کپی می‌شود. بعد از آن، منبع اصلی خودِ Blobs است، نه فایل کد. یعنی اگر بعداً `data/books.js` را در کد عوض کنید، روی سایت واقعی تاثیری نخواهد داشت مگر اینکه از خود پنل استفاده کنید.
